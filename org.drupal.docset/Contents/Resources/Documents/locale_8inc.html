enum LOCALE_JS_OBJECT

Regular expression pattern used to match simple JS object literal.

This pattern matches a basic JS object, but will fail on an object with nested objects. Used in JS file parsing for string arg processing.

enum LOCALE_JS_OBJECT_CONTEXT

Regular expression to match an object containing a key 'context'.

Pattern to match a JS object containing a 'context key' with a string value, which is captured. Will fail if there are nested objects.

locale_string_is_safe ( $  string)

End of "locale-languages-negotiation" Check that a string is safe to be added or imported as a translation.

This test can be used to detect possibly bad translation strings. It should not have any false positives. But it is only a test, not a transformation, as it destroys valid HTML. We cannot reliably filter translation strings on import because some strings are irreversibly corrupted. For example, a & in the translation would get encoded to &amp; by filter_xss() before being put in the database, and thus would be displayed incorrectly.

The allowed tag list is like filter_xss_admin(), but omitting div and img as not needed for translation and likely to cause layout issues (div) or a possible attack vector (img).
